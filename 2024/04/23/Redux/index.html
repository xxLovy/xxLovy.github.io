Why & When to use Redux

This blog is about how to use Redux Toolkit to get started with Redux.

Redux is to help developer manage the global state when developing.

For example, sometimes we have to “raising” state to make two components communicatable

Well, it’s probably not a good example because it’s too simple. Here I quote the Redux tutorial: Redux helps you deal with shared state management, but like any tool, it has tradeoffs. It’s not designed to be the shortest or fastest way to write code. It’s intended to help answer the question “When did a certain slice of state change, and where did the data come from?”, with predictable behavior. There are more concepts to learn, and more code to write. It also adds some indirection to your code, and asks you to follow certain restrictions. It’s a trade-off between short term and long term productivity.

So we want to use Redux only we have to implement a more complex feature.

And our code will be more structural and neat with the use of Redux

Getting started with Redux

As mentioned before, Redux is a global state management system. Therefore, there are three things that we mainly concerned about:

  1. How to store data.
  2. How to mutate data.
  3. How to get data.

Redux App Structure

Redux projects usually consist of three main components:

  1. Slice for defining global states and managing the operations that potentially applied to the state.
  2. Selectors for fetching the global states.
  3. Store for storing global states and exporting operations so that can be accessed in other components. And for applying middlewares.

Take a counter as an example

counterSlice.js

store.js

First we define the initialState of the state that we want to store. This is also the type of the global state. We can think this as

Instead of being in a single componet, this will be in a global state management center where we can access this state everywhere in our components.

Then we create counterSlice with createSlice method that Redux Toolkit provided.

We define the name of the state, the initialState of the state and finally the reducers.

reducer is for mutating the global state that we created. Each reducer has a name, in this case, it’s increment decrement and incrementByAmount. It’s simple to understand, we want to increase a counter everytime when we click the + button so is -. And we want to mutate the counter by the amount that we want so we have incrementByAmount.

And we have to perform the operation we want to the state.

increment: add state by 1.

decrement: minus state by 1.

incrementByAmount: add state by the amount we give.

Then in the end, we want to expose the state using Selectors so we can access the state we created like using useState in a single component.

And also expose the operation we want to operate with the state.

Through store.js we can expose the state and its reducers(which includes the operations we want to make)

Use Redux in other components

After finishing setting up redux, it’s time for us to use that global state in a component.

Counter.jsx

We can use useSelector hook from Redux Toolkit and selectCount that we exported to get the global state that we stored.

Wait, how we can actually mutate the global state?

With the use of useDispatch hook from Redux Toolkit, we can easily perform the operation we want to apply for the global state( in this case increment decrement and incrementByAmount)

As you may notice, when used action.payload we were dealing with incrementByAmount. payload is the parameters that you want to pass to the operation that you want to perform.

using action.payload to extract the parameters so the operation can be done in a expected way. As we click Add Amount button, it will trigger dispatch(incrementByAmount(incrementValue) which will make our state perform state.value += action.payload. And then the count will automatically sync the change we just made.
